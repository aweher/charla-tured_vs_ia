módulo 01

Higiene de borde

Validación RPKI — la primera línea de defensa para tu tabla BGP.

🛡️ 1 servicio · Profile: capa1 · ayuda.la

Routinator RPKI Validator

Validador RPKI de NLnet Labs que descarga y verifica los ROAs (Route Origin Authorizations) de los 5 RIRs. Permite a tus routers BGP validar el origen de cada prefijo anunciado.

🔗 Acceso

ProtocoloPuertoURL
HTTP API + UI 9556 http://localhost:9556
RTR (routers) 3323 routinator:3323
docker — routinator
# Imagen nlnetlabs/routinator:latest # Container routinator # Profile capa1 # Comando $ server --rtr=0.0.0.0:3323 \ --http=0.0.0.0:9556 # Volúmenes ./routinator/data → ~/.rpki-cache
Nota

Requiere routinator-init (init container) que ajusta permisos del cache antes de arrancar.

módulo 02

Visibilidad

Recolección de flows, almacenamiento columnar y dashboards en tiempo real.

📊 4 servicios · Profile: capa2 · ayuda.la

Kafka Message Broker

Broker de mensajes Apache Kafka en modo KRaft (sin ZooKeeper). Actúa como buffer entre Akvorado (productor de flows) y ClickHouse (consumidor). Retiene mensajes por 24 horas.

🔗 Acceso

ProtocoloPuertoNotas
PLAINTEXT 9092 Solo interno (kafka:9092)
CONTROLLER 9093 Solo interno (kafka:9093)
docker — kafka
# Imagen apache/kafka:latest # Container kafka # Modo KRaft (broker + controller) # Cluster ID akvorado-demo-cluster-01 # Retención 24 horas # Volúmenes ./kafka/data → kraft-logs
Sin puertos expuestos al host

Kafka solo es accesible dentro de la red defensa. Los servicios internos lo alcanzan como kafka:9092.

ClickHouse Columnar DB

Base de datos columnar de alto rendimiento, optimizada para consultas analíticas sobre grandes volúmenes de datos de red. Almacena todos los flows que recolecta Akvorado.

🔗 Acceso

ProtocoloPuertoNotas
HTTP API 8123 Solo interno (healthcheck)
Native 9000 Solo interno
docker — clickhouse
# Imagen clickhouse/clickhouse-server:24.3-alpine # Container clickhouse # DB default akvorado # ulimits nofile 262144 (soft/hard) # Volúmenes ./clickhouse/data → /var/lib/clickhouse ./clickhouse/config.d/ → users.d/ (ro)
Base de datos interna

No expone puertos al host. Grafana y Akvorado la consultan internamente por la red defensa.

Akvorado Flow Collector

Colector y visualizador de flujos de red (NetFlow v5/v9, IPFIX, sFlow). Recibe los flows de routers y switches, los almacena en ClickHouse vía Kafka, y ofrece una UI web para análisis.

🔗 Acceso

ProtocoloPuertoURL
Web UI 8081 http://localhost:8081
NetFlow/IPFIX 2055/udp Receptor de flows
sFlow 6343/udp Receptor de sFlow
docker — akvorado
# Imagen ghcr.io/akvorado/akvorado:latest # Container akvorado # Depende de clickhouse (healthy) kafka (healthy) # Comando $ orchestrator /etc/akvorado/akvorado.yaml # Config ./akvorado/config.yaml (ro)

Grafana Dashboards

Plataforma de observabilidad y visualización. Conecta directamente a ClickHouse para crear dashboards de tráfico, análisis de flujos y métricas de red en tiempo real.

módulo 03

Detección

IDS de red, IDS colaborativo y SIEM — detectá amenazas antes de que escalen.

🔍 5 servicios · Profiles: capa3 siem · ayuda.la

CrowdSec IDS Colaborativo

Motor de seguridad open source y colaborativo. Analiza logs del host (SSH, iptables, nginx) y comparte inteligencia de amenazas con la red global de CrowdSec. Detecta y bloquea IPs maliciosas.

🔗 Acceso

ProtocoloPuertoURL
LAPI (REST) 8080 http://localhost:8080
docker — crowdsec
# Imagen crowdsecurity/crowdsec:latest # Container crowdsec # Collections linux sshd iptables nginx # Volúmenes ./crowdsec/data → /var/lib/crowdsec ./crowdsec/config → /etc/crowdsec /var/log → /var/log (ro)
Inteligencia colaborativa

Las colecciones incluyen parsers + escenarios para detectar brute force SSH, port scans y más. La LAPI comparte señales con la comunidad global.

Suricata IDS de Red

Motor IDS/IPS de alto rendimiento que inspecciona tráfico de red en tiempo real. Corre en network_mode: host para capturar paquetes directamente de la interfaz física configurada.

🔗 Acceso

ModoInterfazNotas
Host network eth0 Variable: SURICATA_INTERFACE
Capabilities

Requiere NET_ADMIN, NET_RAW y SYS_NICE para captura de paquetes.

docker — suricata
# Imagen jasonish/suricata:latest # Container suricata # Red network_mode: host # Comando $ suricata -i eth0 # Volúmenes ./suricata/etc → /etc/suricata ./suricata/var-log → logs ./suricata/var-lib → rules/data

Wazuh Indexer SIEM Storage

Motor de búsqueda basado en OpenSearch que indexa y almacena todos los eventos de seguridad recolectados por Wazuh Manager. Permite búsquedas rápidas sobre alertas históricas.

módulo 04

Mitigación

Detección y bloqueo automático de ataques DDoS volumétricos.

⚡ 1 servicio · Profile: capa4 · ayuda.la

FastNetMon DDoS Detection

Detector de ataques DDoS que analiza tráfico de red en tiempo real. Cuando detecta un ataque volumétrico, puede activar RTBH (Remotely Triggered Black Hole) anunciando la community BGP configurada para desviar el tráfico malicioso.

🔗 Acceso

ModoDetalleNotas
Host network network_mode: host Acceso directo a interfaces
RTBH Community 65007:666 Variable: FASTNETMON_RTBH_COMMUNITY
docker — fastnetmon
# Imagen ghcr.io/pavel-odintsov/fastnetmon-community:latest # Container fastnetmon # Red network_mode: host # Capabilities NET_ADMIN, NET_RAW # Volúmenes ./fastnetmon/etc → /etc/fastnetmon ./fastnetmon/var-log → logs
network_mode: host

FastNetMon necesita acceso directo a las interfaces de red del host para capturar y analizar paquetes en tiempo real. No es compatible con redes bridge.

módulo 06

LLM On-Prem

Inteligencia artificial local — modelos de lenguaje sin depender de la nube.

🤖 2 servicios · Profile: llm · ayuda.la

Ollama LLM Runtime

Runtime para ejecutar modelos de lenguaje grandes (LLMs) de forma local. Soporta modelos como Llama, Mistral, Qwen, Gemma y más. Expone una API compatible con OpenAI para integración con herramientas del NOC.

🔗 Acceso

ProtocoloPuertoURL
REST API 11434 http://localhost:11434
GPU opcional

Descomentar el bloque deploy.resources.reservations.devices para usar NVIDIA GPU.

docker — ollama
# Imagen ollama/ollama:latest # Container ollama # Healthcheck $ curl -f http://localhost:11434/ # Volúmenes ./ollama/models → /root/.ollama # GPU (opcional) driver: nvidia, count: all

Open WebUI LLM Interface

Interfaz web tipo ChatGPT para interactuar con los modelos locales de Ollama. Permite crear chats, gestionar modelos, subir documentos para RAG y personalizar system prompts orientados al NOC.
